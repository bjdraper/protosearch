Protosearch Notebook Guide

1 What does this notebook do?

Protosearch takes a set of known reference proteins and a proteome database, then discovers and classifies related proteins using deep-learning embeddings rather than sequence identity alone. It is protein-family agnostic — the crustacean AGPAT survey is the worked example, but any family with a Pfam domain can be substituted.

The notebook walks through six stages: setup → input → HMMER filter → ESM2 embedding → FAISS clustering → phylogenetics & ancestral reconstruction.

Reference probes — UniProt accessions you supply (e.g. AGPAT1–5, PlsC) → data/query_sequences/reference_probes.faa
Your proteome FASTA(s) — per-species .faa files from Drive (331 files for the crustacean run) → data/proteins_raw/combined_input.faa
Length filter + deduplication — keeps proteins 200–500 AA → data/proteins_filtered.faa
HMMER domain filter — keeps only sequences containing the target Pfam domain (e.g. PF01553) → data/hmmer_hits/
ESM2 embedding — each protein → 1,280-dimensional vector using Meta's protein language model → data/embeddings/hits.npy
FAISS KNN index + Leiden clustering — nearest-neighbour graph → community detection → t-SNE visualisation → results/clustering/
Phylogenetics + ASR — MAFFT alignment → FastTree → optional IQ-TREE2 ancestral sequence reconstruction → results/trees/, results/iqtree/

2 Before you start

💡
Use a T4 GPU runtime In Google Colab, go to Runtime → Change runtime type → T4 GPU before running anything. The ESM2 embedding step is 10–30× faster on GPU. Without it, embedding ~1,500 sequences will take several hours instead of 20–30 minutes.

3 Data preparation

Step 1 — Upload your proteome FASTAs

Upload your .faa or .fasta files to Google Drive. You can use one combined file or many per-species files — the notebook merges them automatically.

⚠️
Pre-filtered vs raw proteomes If your FASTAs are already length-filtered (200–500 AA), the filter step in the notebook is a harmless no-op — it will just pass all sequences through. If you are starting from full proteomes, the filter will remove very short fragments and large multi-domain proteins before the expensive HMMER and embedding steps.

Step 2 — Name your survey

In Step 3 of the notebook, set SURVEY_NAME to a short identifier for your analysis (e.g. 'agpat_crustacea', 'kinase_insects'). This name becomes the working folder on Drive and prefixes all output files.

Step 3 — Define your reference probes

In Step 4, replace REFERENCE_PROBES with UniProt accessions for your known protein family members. These are the "queries" used to label clusters and find homologues. A good reference set has 5–25 proteins representing the range of variants you expect to find.

00

Clone repo from GitHub

Downloads the protosearch repository into /content/protosearch and installs Python dependencies from requirements.txt. Run once per Colab session.

Takes ~1 minute. Idempotent — skipped if already cloned.

~1 min CPU only Once per session
Cell
02

Install system tools & Python packages

Installs hmmer, mafft, fasttree via apt, and all Python packages including fair-esm, faiss-cpu, leidenalg, and ete3 via pip.

Takes 5–10 minutes on a fresh Colab session. The ESM2 model weights are downloaded later, at first use.

5–10 min CPU only Once per session
Cell
03

Set working directory ✏ Edit required

Mounts Google Drive and sets the working directory to MyDrive/{SURVEY_NAME}/.

Set SURVEY_NAME to a short identifier for your analysis:
SURVEY_NAME = 'agpat_crustacea'
Instant User config
Cell
04

Define your protein family ✏ Edit required

Defines the reference probe list, Pfam IDs, length filter, clustering parameters, and catalytic motif patterns.

Change for your family:
  • REFERENCE_PROBES — list of (id, UniProt accession, label, colour) tuples
  • PFAM_IDS — one or more Pfam IDs for the HMMER filter
  • MIN_LENGTH / MAX_LENGTH — typical range for your family
  • ALPHAFOLD_TARGETS, CATALYTIC_MOTIFS — optional; can be left as {}
Instant User config
Cell
05

Write config.yaml

Serialises your settings from Cell 04 into a config.yaml file in the working directory. This file drives all subsequent steps and can be manually edited at any time to tune clustering or tree parameters.

Instant CPU only
Cell
06

Download reference probes from UniProt

Fetches the FASTA sequence for each accession in REFERENCE_PROBES using the UniProt REST API and saves them to data/query_sequences/reference_probes.faa. Skipped if the file already exists.

Requires internet access. Takes ~10–30 seconds depending on the number of probes.

<1 min Network I/O
Cell
07

Download Pfam HMM profile

Fetches the HMM profile for each ID in PFAM_IDS from the Pfam database and presses it with hmmpress. Skipped if already downloaded.

<1 min Network I/O
Cell
08

Locate proteome FASTAs on Drive ✏ Set INPUT_FASTAS

Scans Google Drive for .faa and .fasta files, shows a directory-level summary with file counts and sizes, then asks you to confirm the input paths.

The scanner prints a summary like: 
  data/proteins_filtered/
    153 files  |  248 MB total
    glob: 'data/proteins_filtered/*.faa'
  data/crustome_filtered/
    178 files  |  191 MB total
    glob: 'data/crustome_filtered/*.faa'
Set INPUT_FASTAS to the glob patterns shown: 
INPUT_FASTAS = [
    'data/proteins_filtered/*.faa',
    'data/crustome_filtered/*.faa',
]
The cell merges all matched files into a single data/proteins_raw/combined_input.faa (this is what the rest of the pipeline reads). Idempotent — delete combined_input.faa to re-merge from a different set.
1–5 min Drive I/O
Cell
09

Filter by length + deduplicate

Removes proteins shorter than MIN_LENGTH or longer than MAX_LENGTH amino acids, then removes exact-sequence duplicates within the combined FASTA. Writes data/proteins_filtered.faa.

If your FASTAs are already length-filtered, this step passes all sequences through unchanged — it is a safe no-op.

<1 min CPU only
Cell
10

HMMER domain prefilter

Runs hmmsearch against the downloaded Pfam profile. Only proteins with a significant domain hit (default E-value < 1e-5) continue to the embedding step. This dramatically reduces the number of sequences that need to be embedded.

For the crustacean AGPAT run: ~80,000 input sequences → ~1,500 HMMER hits (PF01553). Takes 5–15 minutes depending on proteome size. Controlled by hmmer.cpu in config.yaml.
5–15 min CPU-bound
Cell
11

ESM2 embedding of HMMER hits 🔥 Most expensive step

Loads the ESM2-650M protein language model and computes a 1,280-dimensional embedding for each HMMER hit sequence. These vectors capture evolutionary and structural information far beyond what sequence identity alone provides.

Runtime (1,500 sequences):
 • T4 GPU: ~20–30 minutes
 • CPU only: 3–6 hours

Memory: ESM2-650M requires ~3 GB GPU RAM. Colab T4 has 16 GB — sufficient.

On first run, the model weights (~1.3 GB) are downloaded from HuggingFace. Subsequent runs in the same session reuse the cached weights. Embeddings are saved to data/embeddings/hits.npy — delete to recompute.
20–30 min (GPU) GPU strongly recommended 3 GB GPU RAM
Cell
12

ESM2 embedding of reference probes

Embeds the small reference probe set (typically 15–25 sequences) using the already-loaded ESM2 model. Saved to data/embeddings/ref.npy.

Fast — seconds to a couple of minutes. Skipped if embeddings already exist.

1–5 min GPU (loaded)
Cell
13

Build FAISS KNN index

Constructs a flat L2 nearest-neighbour index over all hit embeddings using FAISS. This enables both fast vector search and the k-NN graph used for Leiden clustering. Saves the index and an ID map to data/knn_index/.

Near-instant for <50,000 vectors. Switch to IndexIVFFlat or IndexHNSWFlat if your dataset grows beyond ~1 million vectors.

<1 min CPU only
Cell
14

Query KNN — find nearest neighbours of reference probes

For each reference probe, finds the k most similar sequences in the FAISS index (default k=50 per probe). Results saved to results/knn_hits.tsv with L2 distance scores.

<1 min CPU only
Cell
15

Distance distribution plot

Diagnostic plot showing L2 distance vs KNN rank for each reference probe. Use this to choose or validate the distance threshold — the point where the curve flattens indicates the edge of the "true neighbour" zone.

Instant Diagnostic
Cell
16

Load embeddings for clustering

Loads the hit and reference embeddings from .npy files into memory in preparation for clustering.

Instant I/O only
Cell
17

Leiden clustering

Runs the full unsupervised clustering pipeline: PCA dimensionality reduction → FAISS k-NN graph construction → Leiden community detection → majority-vote labelling of each community by its nearest reference probe. Saves assignments and a cluster summary to results/clustering/.

Key tunable parameters in Cell 04 / config.yaml:
  • k_neighbors — graph connectivity (default 25; increase for smoother boundaries)
  • resolution — Leiden resolution (higher = more, smaller clusters)
  • pca_dims — PCA components before k-NN graph (default 50)
Also runs t-SNE for 2D visualisation. Takes 2–8 minutes depending on dataset size.
2–8 min CPU-bound
Cell
18

t-SNE cluster plot

Renders the 2D t-SNE projection coloured by Leiden community, with reference probes overlaid. Saved to results/clustering/tsne_clusters.png.

<1 min CPU only
Cell
19

Sub-clustering Optional

Re-runs clustering on a single community of interest to resolve its internal structure. Set SUBCLUSTER to a cluster label (e.g. 'LCLAT1') to activate. Results saved under results/clustering/.

Optional 2–5 min
Cell
20

MAFFT alignment + FastTree

For each cluster of interest, extracts the hit sequences, runs a MAFFT multiple sequence alignment, and builds an approximate ML phylogenetic tree with FastTree (LG substitution model). Trees saved in Newick format to results/trees/{cluster}/.

Time depends heavily on cluster size. With --auto flag:
  • <200 sequences: ~2–5 minutes
  • 200–500 sequences: ~10–20 minutes
  • >500 sequences: consider splitting clusters
5–20 min CPU-bound
Cell
21

IQ-TREE2 ancestral state reconstruction Slow

Runs IQ-TREE2 with bootstrap resampling and ancestral state reconstruction (ASR). Produces a high-confidence tree with branch support values and probability distributions over ancestral amino acid states at each internal node.

Set RUN_IQTREE = True to activate. Controlled by IQTREE_BOOTSTRAP and IQTREE_THREADS.

Runtime per cluster on T4: 30–90 minutes depending on sequence count and bootstrap replicates. IQ-TREE2 is downloaded from GitHub at first run.
Optional 30–90 min Very slow on CPU
Cell
22

Parse ASR + generate plots

Reads the IQ-TREE2 .state output, identifies key ancestral nodes, computes consensus ancestral sequences, and generates three plot types per cluster: sequence logos at variable positions, an ancestral amino acid probability table, and a root confidence / diversity chart. Saved to results/plots/ancestral/{cluster}/.

Optional 2–5 min

6 Customising for your protein family

Minimum changes to run on a new family

  1. Cell 03: SURVEY_NAME = 'your_family_name'
  2. Cell 04: Replace REFERENCE_PROBES with your UniProt accessions
  3. Cell 04: Replace PFAM_IDS with your Pfam domain(s)
  4. Cell 08: Set INPUT_FASTAS to your proteome glob patterns

Finding your Pfam domain ID

  1. Go to InterPro and search for your protein family
  2. Find the relevant Pfam entry (usually labelled Pfam → Family or Pfam → Domain)
  3. Copy the PF number (e.g. PF00069 for protein kinase domain)
🎯
Calibrate your distance threshold Before querying unknowns, check that all your reference probes cluster tightly together in the t-SNE plot (Cell 18). If close family members scatter to different areas, try increasing k_neighbors or decreasing resolution in config.yaml.

8 Troubleshooting

Issue

"CUDA out of memory" during embedding

Lower the batch size in Cell 04: cfg.embedding['batch_size'] = 16 (down from 32). Alternatively, use 'device': 'cpu' — slower, but uses RAM instead of VRAM.

Issue

No FASTA files found by the scanner in Cell 08

Check that you have mounted Google Drive (Cell 03 must have completed successfully). Verify the file is in MyDrive/{SURVEY_NAME}/data/proteins_filtered/. You can also supply an absolute Drive path directly: INPUT_FASTAS = ['/content/drive/MyDrive/my_data/proteome.faa'].

Issue

HMMER finds zero hits

The E-value threshold may be too strict, or the wrong Pfam ID is set. Try relaxing hmmer.evalue to 1e-3 in config.yaml. Also verify your Pfam ID on InterPro — some families span multiple Pfam domains; you may need to add more entries to PFAM_IDS.

Issue

All sequences end up in one cluster

Increase the Leiden resolution parameter (try 1.5 or 2.0). If that doesn't help, try increasing k_neighbors. Re-run from Cell 17 — you don't need to re-embed.

Issue

Colab disconnected mid-run

Reconnect and re-run from Cell 00. All steps that write output files check for existing results and skip recomputation — you will only repeat steps that hadn't finished. The most important files to preserve between sessions are the .npy embedding files, which take the longest to produce.
